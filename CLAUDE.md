⚠️ Spotify API Rate Limit 주의사항 (작업 전 반드시 읽기)

Spotify Web API는 30초 롤링 윈도우 기준으로 호출 수를 제한해요.
초과 시 429 에러가 발생하고, 대시보드에서 확인한 "초기화까지 N분" 동안
모든 Spotify 호출이 실패해요.

이 프로젝트는 현재 development mode (낮은 rate limit) 상태예요.

코드 수정 시 반드시 지켜야 할 것:

1. 테스트용 Spotify 호출 최소화
   - 디버깅할 때 console.log로 중간 결과 확인하고
     실제 API 호출은 꼭 필요한 경우만 실행
   - 불필요한 연속 호출 금지

2. 429 발생 시 Retry-After 헤더 준수
   - 응답 헤더의 Retry-After(초) 만큼 대기 후 재시도

3. 병렬 호출 수 제한
   - Promise.all로 Spotify 검증할 때 후보곡 수 × 호출 수 고려
   - 후보 5개 × 검증 2회 = 10회/요청 → 동시 사용자 늘면 금방 한계 도달

4. 로직 수정 시 실제 API 말고 mock으로 먼저 테스트
   - Spotify 검증 로직 바꿀 때는
     기존 응답 데이터를 하드코딩해서 로직만 먼저 검증
   - API 연동은 로직 확인 후 마지막에

5. 현재 rate limit 상태 확인 방법
   - /admin 대시보드 하단 Spotify API 상태 카드 확인
   - "429 제한 중" 표시면 테스트 중단하고 대기

⚠️ 작업 범위 관련 주의사항

수정해야 할 파일이 3개 이상이거나, 기존 로직을 크게 바꿔야 하는 경우
반드시 나에게 먼저 설명하고 승인을 받은 후 진행해.

승인 없이 대규모 수정을 진행하지 말 것.
