# 🧪 배포 전 스모크 체크리스트

> 주요 배포 후 **5분 안에** 수동으로 확인하는 핵심 유저 flow.
> 자동화 테스트 없는 지금, 이게 유일한 회귀 방지 장치.
>
> **사용법**: 배포 전 로컬 빌드 or 배포 직후 프로덕션에서 1회 끝까지 플레이.
> 실패 항목 1개라도 발견 시 **즉시 롤백 검토**.

## 🔴 필수 flow (매 배포 시)

### 1. 랜딩 페이지 → 사진 업로드
**경로**: `/`

- [ ] 페이지 로드 정상 (흰 화면 / 에러 없음)
- [ ] "Play the Picture" 헤더 + 헤드라인 "오늘 찍은 사진에 어떤 노래가 어울릴까?" 표시
- [ ] **0장 상태**: "사진 추가하기" 핑크 버튼 노출
- [ ] 사진 1장 선택 → 카운트 배지 `1 / 5`로 변경
- [ ] **1장 이상 상태**: 버튼이 "다음" + 화살표 아이콘으로 전환
- [ ] 사진 5장까지 추가 가능 (+ 슬롯이 5장째 사라짐)
- [ ] 업로드한 사진 ✕ 버튼으로 개별 제거 가능
- [ ] 동영상 파일 선택 시 토스트 "사진 파일만 추가할 수 있어요 📷" 노출
- [ ] 사진 3장 이상 시 오른쪽 페이드 그라데이션 보임 (스크롤 힌트)

### 2. 선호 선택 → 분석 시작
**경로**: `/preference`

- [ ] 헤드라인 "정확한 추천을 위해 두 가지만 알려주세요" 표시
- [ ] 장르 섹션: `Music2` 아이콘 + "어떤 음악이 끌려요?" 라벨
- [ ] 분위기 섹션: `SlidersHorizontal` 아이콘 + "어떤 분위기로 듣고 싶어요?" 라벨
- [ ] 장르 8개 + 분위기 4개 선택 가능
- [ ] **둘 다 미선택 시** "분석 시작하기" 눌러도 에러 메시지
- [ ] 선택 후 분석 시작 → 로딩 화면 전환 (사진 with 블러 + "사진 속 오늘을 읽고 있어요 🔍")
- [ ] 2단계 로딩 → "사진 속 오늘을 읽었어요 ✦" + 바이브 스펙트럼 애니메이션
- [ ] 분석 완료 시 `/result`로 자동 이동 (일반적으로 10-15초)

### 3. 결과 화면 렌더링
**경로**: `/result`

- [ ] 상단 "플더픽의 추천곡" 서브 문구 + "Play the Picture" 핑크 로고
- [ ] 사진 썸네일 표시 (1-5장, 가운데 정렬)
- [ ] 썸네일 클릭 시 모달 확대 동작
- [ ] "오늘의 당신은" + vibeType (이모지 + 한글) + vibeDescription 표시
- [ ] 곡명 (굵은 폰트) + 아티스트명 + 태그 3개 (#으로 시작)
- [ ] "플더픽이 추천한 이유" 박스에 reason 텍스트 표시
- [ ] 앨범아트 배경 블러 적용 (매칭 성공 시)

### 4. 30초 미리듣기 재생 ⭐ 오늘의 핵심 기능
**경로**: `/result`

- [ ] 페이지 로드 후 ~1초 내 미리듣기 플레이어 자동 노출 (iTunes 매칭 성공 시)
- [ ] 플레이어는 조약돌 UI (핑크 그라디언트 + 광택)
- [ ] 재생 버튼 (핑크 원형) 탭 → 음악 재생 시작
- [ ] 진행바가 부드럽게 움직임 (뻑뻑함 X, rAF 60fps)
- [ ] 진행바 상단 "30초 들어보기" 라벨 중앙 정렬
- [ ] 오른쪽 시간 `0:XX / 0:30` (elapsed / total) 포맷 정상
- [ ] **8초 경과 후** "음악앱에서 듣기" 핑크 CTA 등장 (페이드 인 애니메이션)
- [ ] 재생 중 일시정지 → 재생 버튼 다시 나타남, 그 후 재개 가능
- [ ] **일시정지 상태에서도 8초 타이머 계속 작동** (이전엔 안 됐음, 회귀 방지)
- [ ] 30초 완주 후 진행바 0으로 리셋 + 다시 재생 가능
- [ ] iTunes 매칭 실패 시 플레이어 미노출, "음악앱에서 듣기" 바로 표시

### 5. 음악앱에서 듣기 (딥링크)
**경로**: `/result` → 바텀시트

- [ ] "음악앱에서 듣기" 클릭 시 바텀시트 등장
- [ ] 상단 핑크 뱃지에 "곡명 — 아티스트" 표시
- [ ] "🎵 링크 찾는 중..." 로딩 → 링크 로드 완료
- [ ] Spotify 버튼 클릭 시 Spotify 앱/웹에서 곡 재생 (deep link 정상)
- [ ] YouTube Music 버튼 클릭 시 YouTube Music 앱/웹에서 곡 재생
- [ ] "바로 재생" vs "검색 화면으로 이동" 레이블 정상 노출
- [ ] "나중에 듣기" 클릭 시 바텀시트 닫힘

## 🟡 핵심 flow (주요 변경 시)

### 6. 보관 · 공유 기능
**경로**: `/result` 하단 버튼

**보관하기:**
- [ ] `Bookmark` 아이콘 + "보관하기" 라벨
- [ ] 클릭 → `entries` 테이블에 insert + `save_logs` 기록
- [ ] 성공 시 버튼이 "보관됨" (`Check` 아이콘 + 핑크 컬러)로 변경
- [ ] 토스트 "✦ 오늘의 기록이 저장됐어요 · 모아보기 →" 노출, 클릭 시 `/journal`로 이동
- [ ] 2번 이상 클릭 불가 (잠금 처리)

**공유하기:**
- [ ] `Camera` 아이콘 + "결과 공유하기" 라벨
- [ ] **모바일**: Web Share API 실행 → OS 공유 시트 열림
- [ ] **데스크탑**: 클립보드 자동 복사 → "링크 복사됐어요!" 토스트
- [ ] 클립보드 실패 시 URL 직접 보여주는 모달 (fallback)
- [ ] 공유 URL 형식: `https://play-the-picture.vercel.app/share/{entryId}`

### 7. 아카이브 (`/journal`)
**경로**: 하단 탭 "아카이브"

- [ ] 보관된 기록 카드 형태로 리스트 표시
- [ ] 각 카드에 사진 + 곡명 + vibeType + 날짜
- [ ] 카드 클릭 → 상세 시트 (또는 결과 유사 뷰)로 이동
- [ ] 보관 기록 없을 때 빈 상태 메시지 정상
- [ ] 하단 "노래 추천받기" 탭 클릭 시 랜딩으로 복귀

### 8. 공유 URL (`/share/[id]`)
**경로**: 누군가 공유한 링크 접속 시

- [ ] URL 직접 접속 시 결과 정보 렌더링 (해당 entry의 vibeType·곡·사진)
- [ ] "나도 해보기" CTA 노출, 클릭 시 랜딩으로 이동
- [ ] OG 이미지 정상 생성 (카톡·트위터 미리보기 확인, 스모크는 URL fetch만)
- [ ] 존재하지 않는 id 접근 시 에러 또는 fallback 화면
