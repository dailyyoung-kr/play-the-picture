## 1. 서비스 정체성 (영구)

### 한 줄
**"사진 한 장으로 오늘의 한 곡"** — 시각 맥락 기반 음악 추천 + viral toy

### 카테고리 정의
시장의 모든 음악 추천은 "내가 들었던 것" 기반. 플더픽은 **"지금 이 순간의 시각 맥락"** 기반 새 카테고리.
가장 가까운 모델: Lensa(AI 사진 후크) + Spotify Wrapped(음악+공유) + MBTI(자기 정체성)의 혼합형.

### 타겟
18~24 한국 여성 (확장 가능)
- 매일 사진·음악 소비, 일기 귀찮음
- 분위기 안 맞는 음악 싫음
- 단톡방·인스타 공유 친화적

### 차별화 4축
1. 시각 input → 음악 output (텍스트 입력 X)
2. vibeType = MBTI 밈 구조의 자기 정체성 콘텐츠
3. 1곡만 추천 (선택 피로 제거)
4. 18-24 한국 큐레이션 (1,365곡 DB)

### 브랜드 컬러 / 톤
- 다크 #0d1218 + 로즈핑크 #C4687A
- 톤: 감성적이지만 과하지 않은, 미니멀, 약간의 위트

## 4. 의사결정 히스토리 (M1~M6, "왜 이 결정")

### M1 · 4/9~14 — 추천 시스템 안정화
**WHY**: 1차 배포 후 Spotify API 18회 호출이 일 할당량 즉시 소진. 검증 실패 다발.
**결정**: 호출 18→3회, Claude 자가검증 STEP 1~3, 1곡 응답으로 단순화.
**학습**: AI 응답 hallucination은 외부 검증으로 잡되, 검증 횟수는 비용·속도 trade-off.

### M2 · 4/15~17 — Opus 4.7 전환 + 대시보드
**WHY**: Sonnet의 톤이 약함. 결과 화면 vibeType 매력 부족 → 공유 0%.
**결정**: Opus 4.6 → 4.7로 모델 업그레이드. 대시보드 8단 funnel 정의·시각화.
**학습**: AI toy의 본질은 콘텐츠 톤. 모델 비용 ↑이라도 톤 차이가 viral 좌우.

### M3 · 4/18~20 — vibeType 정체성 + 첫 viral
**WHY**: 4/17 분석 결과 듣기 클릭이 강해 *"Spotify 포털"*화. 결과 화면 가치 미전달.
**결정**: vibeType 프롬프트 "별명형(수집가/탐험가/요정)" → "정체성형(과몰입러/제조기/설계자)"으로 교체.
**결과**: 4/18 저녁 8시 첫 viral 5건 (CPA ₩221).
**학습**: 저장형(정체성) vs 공유형(풍경 관찰) vibeType 톤이 다름. 같이 가지 않음.

### M4 · 4/22~23 — 30초 미리듣기 도입
**WHY**: 듣기 74%가 외부 앱 이탈로 직결. 결과 화면 체류 8초.
**결정**: iTunes Search API로 30초 미리듣기 + 8 Phase UX 이터레이션.
**결과**: **공유율 2.3% → 15.4% (+570%), 저장율 7% → 19.2% (+174%)**.
**학습**: "외부 이탈 늦추는 gate"가 retention·viral 동시 개선 가능.

### M5 · 4/24~27 — 단톡방 톤 + 도메인
**WHY**: og:title을 곡명으로 했더니 공유 전환 0%. og 카드 메타 검증 필요.
**결정**: og:title을 vibeType으로 복귀. playthepicture.com 도메인. 프롬프트 톤을 "1인칭 공감 → 3인칭 단톡방 시추에이션"으로 전환.
**결과**: 4/27 두번째 viral spike (📸 셀카장인 13v·9u, 28h 지속).
**학습**: vibeType이 viral 핵심. 공유 메시지는 "단톡방 끝까지 읽을 톤"으로 작성.

### M6 · 4/29~30 — 미리듣기 인프라 완성
**WHY**: iTunes 매칭률 73%로 신곡 25%가 미리듣기 X.
**결정**: 자동 매칭(Duration·LLM) + Manual 매칭으로 99.7% 도달.
**부가 발견**: iTunes Search explicit 글로벌 차단 / Apple Music K-시장 곡명 상이 패턴.
**학습**: 외부 API 정책 변화 즉각 영향. iTunes·카톡·Spotify 모두 모니터링 필요.

## 10. 외부 API 정책 박제 (4월 발견)

### iTunes Search API
- explicit (19금) 곡 글로벌 차단 (검색 결과 제외)
- `country=us`/`jp` 폴백, `&explicit=Yes` 모두 무효
- `lookup?id=...`는 ID 명시 시 정상
- → 검색 흐름으로는 explicit 곡 자동 매칭 불가 (manual 필수)

### Apple Music K-시장 곡명 상이 패턴
- Spotify(영문) ↔ Apple Music KR(한글) 곡명 자체 다름
- 예: Countdown! / 마음 따라 뛰는 건 멋지지 않아? — 투어스
- → ISRC 매핑 또는 manual 필수

### Spotify Preview API (2024 deprecated)
