# 플더픽 개발 방향성 (우선순위) 설정

> 5/11 박제. 광고 burn vs 수익화 검증 vs 카테고리 본질 고민 + 3가지 옵션 분석 + 결정.

---

## 1. 처음 검토한 3가지 방향

### 공유 기능 개선
서비스의 본질은 사진에 맞는 노래를 추천해주는 것이지만,
결국 서비스가 활성화되기 위해서 필요한 것은 자발적인 공유를 통한 바이럴.

### iOS 앱 개발
웹앱이라는 방식 특성상 리텐션 및 체류 시간을 늘리기 어려운 구조.
앱 출시 및 간편 로그인 도입 등을 통해 [저장] 또는 [재방문] 했을 때의 만족도를 올리고,
네이티브 앱이 아니어서 생기는 여러 기능적인 제한 (특히 인스타그램 인앱에서의 공유 기능 한계) 를 해결.

### 수익화 가능성 검토
지금 메타 광고에 의존해 고객 1명을 분석시키는 데 약 500원의 비용이 들어감.
(CAC 300~400원 + 평균 분석 2~3회 × API 호출 30원 = 약 500원)

수익화 가능성을 검증하지 않은 상태에서는 DAU가 늘어나도 유지가 갈수록 어려워지는 구조.
최소한 BEP라도 확보할 수 있겠다는 가능성을 확인해본 후에 추가적인 기능 개발이 합리적.

## 2. 데이터 기반 3가지 옵션 분석

### Option 1: Viral 활성화
- 이미 31% — **카테고리 best-in-class**
- 추가 lever marginal
- try_click 트렌드 -44% (행동 ↑ but 결과 ↓)
- ❌ ROI 낮음

### Option 2: 곡 추천 품질·정체성 강화
- 듣기 84% — 이미 압도적
- positioning value (B2B pivot·콘텐츠 매개)
- ⚠️ marginal product upside, 큰 brand upside

### Option 3: ⭐ 저장/기록 앱 정체성 (선택)
- 7% → 30% 도달 시 **카테고리 자체 이동** (#5 Entertainment → #3 Niche utility)
- D7 retention 본질적 한계 돌파 시도
- 가장 큰 upside + 가장 큰 risk

→ **결정: Option 3 (70%) + Option 2 (30%)**
