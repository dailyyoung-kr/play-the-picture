## 핵심 발견 (data-driven)

### 1. Race condition 확정 (rapid stress test 한정)
- `new-recommend.ts:599` STEP 4 INSERT는 `after()` 비동기
- 5초 이내 연속 호출 시: 호출 3의 history INSERT 완료 전 호출 4가 read → prefix 차단 실패
- 일반 prod (호출 간 분~시간): 영향 없음
- **결정**: 그대로 두기 (A1 옵션 — sync insert로 변경하는 건 +60~100ms 모든 사용자 비용 vs ~1% 사용자 효익)

### 2. Stress test 회귀 패턴 (3 데이터셋)
| 사진 유형 | vibeType prefix 다양성 | suffix 회귀 |
|---|---|---|
| 인생네컷 흑백 6장 (1장 반복) | 5/6 (race condition 추정) | 삼인조 2/6 |
| 셀카 5장 (1장 반복) | 5/5 (100%) | 장인 3/5 (60%) |
| 기차 4장 (멀티) | 4/4 (100%) | 수집가 2/4 (50%) |

→ **prefix block은 작동**, **suffix 회귀(~50%)는 사진 유형 무관 일반 패턴**

### 3. 회귀 좌표는 차단하면 이동 (whack-a-mole)
- dominant 디테일 차단 → describable 부차(소품)로 이동
- 폰케이스(소품) 차단 → 셀카 행위 메타("장인")로 이동
- prefix 차단 → suffix 회귀
- **함의**: 같은 사진 반복 호출의 100% 다양성은 prompt만으로 불가. server-side suffix block까지 가야 함.

## 운영 인사이트 (메모)

### Cold Opus가 시간 잡아먹는 패턴
1. **콤마 나열 vs bullet**: bullet이 attention 보전에 압도적 우위 (실증)
2. **모호한 슬래시 `/`**: "또는" vs "리스트" 모호 → 모델은 안전하게 "다 포함" 회귀
3. **노이즈 instruction** ("매번 다르게" in single-call): 모델 reasoning 단계 추가
4. **중복 룰**: 같은 룰 여러 곳 등장하면 모델은 "왜 2번?" 잠깐 reasoning

### 위계 마커(⚠️/금지) 효용
- 실측 결과 마커 차이는 큰 효과 없음 — 모델은 둘 다 비슷한 강도로 받음
- 단일 마커로 통일하는 게 인지부하 ↓
- ⚠️ 인플레이션 (3개 이상) 시 의미 약화

### 톤 풀 길이 편차
- 자기자랑만 4문장(150자), 나머지 1문장(~30자) → 자기자랑 받으면 토큰·처리 시간 ↑
- 압축 후에도 자기자랑 톤 결 유지됨 (prod 카드 4의 "예약각 아닐까요?" 살아있음)
