## 4. Apple 로그인 측정 버그 fix (commit 0623abc)

### 4-1. 진단

[src/app/auth/callback/route.ts:216](../src/app/auth/callback/route.ts:216)에서 OAuth 신규 가입 시 `event: "google_login_success"` 하드코딩. 5/14 Apple 추가 시 callback 수정 누락.

증거: 5/14~5/17 사이 `apple_login_start` 직후 1분 내 `google_login_success`로 끝나는 device 16건 — 모두 진짜 Apple 가입자가 google로 잘못 분류됨.

### 4-2. 수정

```ts
const provider = exchangeData?.user?.app_metadata?.provider ?? "google";
const loginEvent = provider === "apple" ? "apple_login_success" : "google_login_success";
await adminClient.from("auth_logs").insert([
  { device_id: deviceId, user_id: userId, event: loginEvent },
  ...
]);
```

### 4-3. 과거 보정 (사용자가 Supabase Dashboard에서 직접 실행 완료)

16건 UPDATE — `apple_login_start` 후 5분 내 `google_login_success` 패턴 device의 row를 `apple_login_success`로 갱신 + metadata에 `corrected_from`/`corrected_at` 박제.

### 4-4. 결과

신규 Apple 가입부터 정확히 기록. 5/14 이후 Google/Apple 실제 분포는 약 17:17 반반 (이전 통계 34:1은 잘못된 분류).

## 5. Claude prompt 4톤 시스템 (commit e95a024) ⭐⭐⭐

오늘 가장 큰 작업. 위치: [src/app/api/analyze/new-recommend.ts](../src/app/api/analyze/new-recommend.ts).

### 5-1. 문제 발견 패턴

테스트 케이스 (`f859afd6` 5/17 헤비유저, 같은 사진 32회 분석):
- vibe_type "🖤 흑백 ___" 18연속
- vibe_description 토씨 동일 반복 ("프로필 사진 후보만 47장째" 5번)
- 결정론적 패턴 회귀

데이터 검증:
- 어미 편향: 수집가 17.8% + 러/러버 28% = 45.8% 두 어미 몰림
- "~중" 종결 19.7%
- 5/13 "수집가 편향 완화" commit 효과 0
- LIKED 검증된 어미: ~러 (40%), 장인, 모델, 주인공, 사장님

### 5-2. 개선 원칙 — Negative constraint 대신 Positive force

stateless prompt에서 "반복하지 말 것"은 작동 안 함. 매 호출 외부에서 옵션 1개를 force하는 방식.

### 5-3. 변경 요약

**SYSTEM_PROMPT** — 픽터 캐릭터 정체성 + 사진(최대 5장) + 단톡방·인스타 공유 컨텍스트:

```
'플더픽(Play the Picture)'은 사진을 올리면(최대 5장) 분위기를 읽어서 어울리는 음악 1곡을 추천해주는 서비스야.
분석 카드를 단톡방·인스타에 띄워서 친구들끼리 돌려보며 한 마디씩 던지는 게 가장 즐기는 방식이야.
너는 그 서비스의 마스코트 '픽터' — 친구처럼 친근한 캐릭터.
사진을 분석해서 후보곡 중 어울리는 음악 1곡을 골라.
톤은 친구처럼 친근하게 — 위트·농담·호들갑 OK. 감성적·잔잔한 톤은 픽터와 안 맞음.
reason은 존댓말(~요체)이지만 친구가 카톡으로 말하는 듯한 자연스러운 호흡.
vibeDescription은 명사로 종결 — 친구 단톡방 카피처럼 짧고 임팩트 있게.
한국어, JSON만 응답.
```

**VIBE_TONE_POOL** — 4톤. 매 호출 1개 랜덤 + 그 톤의 format_examples 풀(7~8개)에서 3개 랜덤 추출:

| 톤 | 결 | desc_examples (LIKED 검증) |
|---|---|---|
| **농담관찰** | 삐딱한 친구 농담 + 자조 | 잔디밭이 무대인 줄 아는 사람 / 필기 3줄 쓰고 폰 30분 보는 중 / 산 전세 내고 라떼 한 잔 |
| **자기PR** | 단톡방 자기 자랑 + 자뻑·호들갑 | 사진 찍는 자세 하나로 웃음 바다 만드는 폼 미쳤음 / 이 구역의 분위기 메이커는 바로 나 / 힙스터라는게 별게 있나? |
| **직설인정** | 짧고 단호한 친구 말투 + 인정 | 조명 끄고 감성 켜는 시간대 / 선인장한테 비니 씌우는 감각 / 어금니가 스크린 데뷔하는 날 |
| **유쾌발견** | 친구가 디테일 발견 + 위트 | 히라가나보다 아이스크림이 먼저 녹는 중 / 발밑 풍경도 작품이 되는 사람 / 언니 손목시계가 오늘의 주인공 |

**vibeType 가이드** — 어미 자유 + 풀 예시 + ~러 금지:
```
어미는 사진 소재와 자연스럽게 매칭되는 직업·역할·정체성 자유 선택 —
예: 요정/모델/주인공/탐험가/팬/스타일리스트/감독/사장님/단골/사색가/관찰자/

'~러' 어미 사용 금지 (사진 소재 + 러 패턴 편향 회피).
```

```

```

```

```

- reason 1번째 문장은 40자 이내로 간결하게

## 6. 로딩 UX 개선 (commit a76b7c8 + RN e964ef7)

### 6-1. 진단

평균 분석 시간 11.9초 (p50 11.5s, p95 15.2s, p99 19.5s). PHASE3 메시지가 3초 간격 6개 → 평균 유저는 1~2개만 노출. 더 큰 문제:

```
6~9초:  "딱 맞는 한 곡을 고르고 있어요"
9~12초: "거의 다 골랐어요"    ← 마무리 인상
12~15초: "픽터가 플레이리스트를 뒤적이는 중..."  ← 역행 (인지 부조화)
```

### 6-2. 수정 (web + RN 동일)

**메시지 순서 재정렬** (자연 진행):
```
1. 픽터가 플레이리스트를 뒤적이는 중...   (시작)
2. 딱 맞는 한 곡을 고르고 있어요
3. 다른 곡과 한번 더 비교하는 중
4. 한 곡인 만큼 신중하게
5. 거의 다 골랐어요                      (마무리 직전)
6. 마지막으로 한번 더 확인해볼게요        (정지)
```

**간격 단축** 3000ms → 2000ms. 평균 유저 메시지 1~3, p95 1~5, p99 1~6 노출.

### 6-3. RN 동기화 추가 변경

기존 RN에 "플더픽이 플레이리스트를 뒤적이는 중..." 그대로 남아 있던 거 → "픽터가"로 동기화 (web 664c489와 짝).
